\begin{abstract}
This document provides an example of using Sweave 
to record console input and output.
Such a format is useful for several purposes including
R tutorials, informal analyses, and analyses that will be consumed
by readers knowledgeable about a specific project and about R.
The example involves performing an item analysis of 
responses of 127 students 
to a set of 50 multiple choice test items.
The test items were developed by students as part of an informal
class exercise.
They are by design of widely varying quality and difficulty.
For a copy and explanation of the source code, go to
\url{http://jeromyanglim.blogspot.com/2010/11/sweave-tutorial-3-console-input-and.html}
\end{abstract}

\section{Simple Attempt to Improve Scale}
\subsection{Removal of outlier cases}
\begin{knitrout}
\definecolor{shadecolor}{rgb}{0.969, 0.969, 0.969}\color{fgcolor}\begin{kframe}
\begin{alltt}
\hlstd{orcases} \hlkwb{<-} \hlstd{cases[}\hlopt{!}\hlstd{cases}\hlopt{$}\hlstd{id} \hlopt{%in%} \hlstd{outlierIds, ]}
\hlstd{oritemstats} \hlkwb{<-} \hlkwd{score.multiple.choice}\hlstd{(}\hlkwc{key} \hlstd{= items}\hlopt{$}\hlstd{correct,}
                \hlkwc{data} \hlstd{= orcases[,items}\hlopt{$}\hlstd{variable])}
\end{alltt}


{\ttfamily\noindent\color{warningcolor}{\#\# Warning in cor(items, scores, use = "{}pairwise"{}): the standard deviation is zero}}\end{kframe}
\end{knitrout}

With the outlier cases the scale reliability was estimated to be 
0.68
with the outlier cases removed scale reliability 
was estimated to be 0.48
The lesson to be learnt here is that failure to remove outlier
cases can lead to a gross overestimation of the reliability of a scale.

\subsection{Removal of poor items}
There are many ways of identifying poor items.

\begin{knitrout}
\definecolor{shadecolor}{rgb}{0.969, 0.969, 0.969}\color{fgcolor}\begin{kframe}
\begin{alltt}
\hlstd{rules} \hlkwb{<-} \hlkwd{list}\hlstd{(}
                \hlkwc{tooEasy} \hlstd{=} \hlnum{.95}\hlstd{,}
                \hlkwc{tooHard} \hlstd{=} \hlnum{.3}\hlstd{,}
                \hlkwc{lowR} \hlstd{=} \hlnum{.15}\hlstd{)}
\hlstd{oritemstats}\hlopt{$}\hlstd{item.stats}\hlopt{$}\hlstd{lowR} \hlkwb{<-}
                \hlstd{oritemstats}\hlopt{$}\hlstd{item.stats}\hlopt{$}\hlstd{r} \hlopt{<} \hlstd{rules}\hlopt{$}\hlstd{lowR}
\hlstd{oritemstats}\hlopt{$}\hlstd{item.stats}\hlopt{$}\hlstd{lowR[}\hlkwd{is.na}\hlstd{(oritemstats}\hlopt{$}\hlstd{item.stats}\hlopt{$}\hlstd{lowR)]} \hlkwb{<-} \hlnum{TRUE}
\hlstd{oritemstats}\hlopt{$}\hlstd{item.stats}\hlopt{$}\hlstd{tooEasy} \hlkwb{<-}
                \hlstd{oritemstats}\hlopt{$}\hlstd{item.stats}\hlopt{$}\hlstd{mean} \hlopt{>} \hlstd{rules}\hlopt{$}\hlstd{tooEasy}
\hlstd{oritemstats}\hlopt{$}\hlstd{item.stats}\hlopt{$}\hlstd{tooHard} \hlkwb{<-}
                \hlstd{oritemstats}\hlopt{$}\hlstd{item.stats}\hlopt{$}\hlstd{mean} \hlopt{<} \hlstd{rules}\hlopt{$}\hlstd{tooHard}

\hlstd{oritemstats}\hlopt{$}\hlstd{item.stats}\hlopt{$}\hlstd{baditem} \hlkwb{<-}
                \hlkwd{with}\hlstd{(oritemstats}\hlopt{$}\hlstd{item.stats,}
                                \hlstd{(lowR} \hlopt{|} \hlstd{tooHard} \hlopt{|} \hlstd{tooEasy))}

\hlstd{baditems} \hlkwb{<-} \hlkwd{row.names}\hlstd{(oritemstats}\hlopt{$}\hlstd{item.stats[}
                                                \hlstd{oritemstats}\hlopt{$}\hlstd{item.stats}\hlopt{$}\hlstd{baditem, ])}
\hlstd{gooditems} \hlkwb{<-} \hlkwd{row.names}\hlstd{(oritemstats}\hlopt{$}\hlstd{item.stats[}
                                                \hlopt{!}\hlstd{oritemstats}\hlopt{$}\hlstd{item.stats}\hlopt{$}\hlstd{baditem, ])}
\end{alltt}
\end{kframe}
\end{knitrout}

The code above uses some simple heuristics to flag bad items.
Items were flagged as bad based on the following rules:

\begin{itemize}
\item \emph{Too Easy}: mean correct $>$
0.95.
11
items were bad by this definition.

\item \emph{Too Hard}: mean correct $<$ 
0.3.
0
items were bad by this definition. 

\item \emph{Low Item--Total Correlation}: item total correlation $<$ 
0.15.
15 
items were bad by this definition. 
\end{itemize}

Overall, these three rules flagged 
21 of 
50 items as bad.

The Spearman Brown prophecy formula provides a means of
estimating the number of items required to achieve a given alpha. 

\begin{knitrout}
\definecolor{shadecolor}{rgb}{0.969, 0.969, 0.969}\color{fgcolor}\begin{kframe}
\begin{alltt}
\hlstd{sbrown} \hlkwb{<-} \hlkwd{list}\hlstd{()}
\hlstd{sbrown}\hlopt{$}\hlstd{targetAlpha} \hlkwb{<-} \hlnum{.8}
\hlstd{sbrown}\hlopt{$}\hlstd{actualAlpha} \hlkwb{<-} \hlstd{reditemstats}\hlopt{$}\hlstd{alpha}
\hlstd{sbrown}\hlopt{$}\hlstd{multiple} \hlkwb{<-} \hlstd{CTT}\hlopt{::}\hlkwd{spearman.brown}\hlstd{(sbrown}\hlopt{$}\hlstd{actualAlpha ,} \hlnum{.8}\hlstd{,} \hlstr{"r"}\hlstd{)}\hlopt{$}\hlstd{n.new}
\hlstd{sbrown}\hlopt{$}\hlstd{refinedItemCount} \hlkwb{<-} \hlkwd{nrow}\hlstd{(reditemstats}\hlopt{$}\hlstd{item.stats)} \hlopt{*} \hlstd{sbrown}\hlopt{$}\hlstd{multiple}
\hlstd{sbrown}\hlopt{$}\hlstd{totalItemCount} \hlkwb{<-} \hlkwd{nrow}\hlstd{(itemstats}\hlopt{$}\hlstd{item.stats)} \hlopt{*} \hlstd{sbrown}\hlopt{$}\hlstd{multiple}
\end{alltt}
\end{kframe}
\end{knitrout}

The formula suggests  that in order to obtain
an alpha of 0.8,
3.55 times as many items are required.
Thus, the final scale would need around
103 items.
Assuming a similar number of good and bad items,
this would require an initial pool of around
178 items.  
It should also be noted that these are probably under estimates
given (a) the relatively small sample size, 
and (b) item total correlations
and alpha are likely to be positively biased due to the selection 
procedure used for identifying good test items.

If you get a bunch of students 
and give them five minutes to write a bunch of random test items,
don't expect the resulting scale to have
good psychometric properties.  
But of course, this document was intended more as an example of 
using Sweave than as an example of best practice in test construction.
